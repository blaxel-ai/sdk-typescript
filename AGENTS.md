# Blaxel TypeScript SDK

Monorepo for the Blaxel TypeScript SDK. The main package is `@blaxel/core`, with framework integrations in `@blaxel/langgraph`, `@blaxel/llamaindex`, `@blaxel/mastra`, `@blaxel/vercel`, and `@blaxel/telemetry`.

## Repository structure

```
@blaxel/core/              Main SDK package
  src/
    client/                Auto-generated controlplane API client (types.gen.ts, sdk.gen.ts)
    sandbox/
      client/              Auto-generated sandbox API client (types.gen.ts, sdk.gen.ts)
      drive/               Drive mount/unmount operations
      filesystem/          File read/write/search operations
      network/             Port proxying (fetch)
      process/             Process exec/logs/streaming
      codegen/             Code editing (fastapply, reranking)
      preview.ts           Preview management (control plane)
      session.ts           Session management (control plane)
      sandbox.ts           SandboxInstance - main entry point
      interpreter.ts       CodeInterpreter (Jupyter)
      system.ts            System upgrade
      types.ts             Shared types (SandboxConfiguration, SessionWithToken, etc.)
    common/                Settings, logger, auth, H2 pooling
    drive/                 DriveInstance (control plane CRUD)
    volume/                VolumeInstance (control plane CRUD)
    image/                 ImageInstance (image builds)
    agents/                Agent helpers
    models/                Model gateway integrations
    tools/                 MCP tool helpers
tests/
  integration/sandbox/     Integration tests (requires BL_WORKSPACE + BL_API_KEY)
  benchmarks/              Performance benchmarks
  e2e/                     End-to-end tests
  runtime-environments/    Cross-runtime compatibility (Node, Bun, Deno, browser, etc.)
```

## Code conventions

### TypeScript
- ESM modules (`"type": "module"` in package.json)
- Target: ES2022, Module: NodeNext
- Use `import type` for type-only imports
- Use `type` over `interface` for type definitions
- No `console.log` in library code

### Auto-generated code
- Files ending in `.gen.ts` are auto-generated by `@hey-api/openapi-ts@0.66.0`
- **Never edit `.gen.ts` files manually** -- they are overwritten on regeneration
- Regenerate with `make sdk-sandbox` or `make sdk-controlplane` (see `/regenerate-sdk` skill)
- Wrapper code (e.g. `drive.ts`) should import and call functions from `sdk.gen.ts`, not make raw HTTP calls

### Sandbox subsystem pattern
Every sandbox subsystem follows this pattern:
```typescript
import { SandboxAction } from "../action.js";
import { someGeneratedFunction, type SomeType } from "../client/index.js";

export class SandboxFoo extends SandboxAction {
  async doSomething(): Promise<SomeResponse> {
    const { response, data, error } = await someGeneratedFunction({
      baseUrl: this.url,
      client: this.client,
      // ... params
    });
    this.handleResponseError(response, data, error);
    return data as SomeResponse;
  }
}
```

Key points:
- Extend `SandboxAction` for access to `this.url`, `this.client`, `this.handleResponseError()`
- `this.url` respects `forceUrl` (for local dev / session-based access)
- `this.client` handles auth, H2 session reuse, and interceptors
- Do NOT use `h2Fetch` directly or construct headers manually with `settings.headers`

### Exports
- All public types and classes must be re-exported through `src/sandbox/index.ts` and `src/index.ts`
- Watch for name conflicts between `client/types.gen.ts` (API types) and SDK classes (e.g. `SandboxNetwork` type vs `SandboxNetwork` class) -- use named exports to resolve

### Testing
- Integration tests require `BL_WORKSPACE` and `BL_API_KEY` environment variables
- Use helpers from `tests/integration/sandbox/helpers.ts` (uniqueName, defaultLabels, defaultImage, etc.)
- Always label test resources with `defaultLabels` for automatic cleanup
- Always clean up sandboxes in `afterAll`
- Run with: `npx vitest run tests/integration/sandbox/<file>.test.ts --reporter=verbose`
- Rebuild before testing: `cd @blaxel/core && npm run build`

### Building
```bash
cd @blaxel/core && npm run build    # builds CJS + ESM + browser variants
```

The build outputs to `dist/cjs/`, `dist/esm/`, `dist/cjs-browser/`, `dist/esm-browser/`. Tests and other packages import from the compiled output, not from source.
